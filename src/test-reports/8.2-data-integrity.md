### 3. Encryption Verification

- **Vault entries**: All password and notes data encrypted via AES-256-GCM with PBKDF2 key derivation (600K iterations). Every read operation calls `assertUnlocked()` before accessing data. Passwords stored as `passwordCiphertext` (blob) + `passwordIv` (text). Notes stored similarly.
- **VCC data**: Only `last_four` digits stored - no full card numbers. No encryption needed.
- **CryptoService features**: 5-minute idle auto-lock, activity tracking (mousemove/keydown/click/scroll/touch), base64 salt serialization, consistent password hashing.
- **Wrong password handling**: `crypto.subtle.decrypt` throws `OperationError` when wrong key is used. The vault repo catches this at the call site level.

### 4. Foreign Key Relationships

| FK | Parent | Child | Cascade | Risk |
|----|--------|-------|---------|------|
| vccs.linked_account_id | emails.id | vccs | NONE | LOW - SQLite FK enforcement off by default. Orphaned VCCs won't crash the app, just show null linked account. |
| inventory_items.order_id | orders.id | inventory_items | NONE | LOW - Same as above. Inventory items with deleted order just show no linked order. |

**Note**: SQLite does not enforce foreign keys by default (requires `PRAGMA foreign_keys = ON`). The app does not enable this pragma, so foreign key violations won't cause errors. Orphaned references are handled gracefully - null/missing parent data is displayed as empty.

### 5. CSV Parser Analysis

- Handles quoted fields with embedded commas correctly
- Handles escaped double quotes (`""`) correctly
- Preserves unicode and special characters
- **Known limitation**: Does not handle embedded newlines within quoted fields. The parser splits on `\n` first, so `"line1\nline2"` in a field would be split into two rows. This is acceptable for the expected use case (bank CSVs, which rarely have embedded newlines).

### 6. Type Safety

All repositories use Drizzle's `InferSelectModel`/`InferInsertModel` for type derivation. The `Omit<>` patterns on create/update functions correctly exclude auto-generated fields (`id`, `createdAt`, `updatedAt`).
