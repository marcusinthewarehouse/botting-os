## 4. Webhook Parser Edge Cases - Bugs Found

### Bug 1: `parsePrice()` only removes first comma
- **File**: `workers/webhook-proxy/src/parsers/utils.ts:6`
- **Code**: `.replace(',', '')` only removes the first comma
- **Impact**: Prices like `$1,000,000.00` parse as `$1000` (100000 cents) instead of `$1000000` (100000000 cents)
- **Real-world impact**: LOW - sneaker prices rarely exceed $99,999
- **Fix**: Change `.replace(',', '')` to `.replaceAll(',', '')` or `.replace(/,/g, '')`

### Bug 2: `normalizeFieldName()` matches empty string to 'product'
- **File**: `workers/webhook-proxy/src/parsers/normalize.ts:25-27`
- **Code**: Fuzzy fallback uses `a.includes(lower)` - every string includes `''`
- **Impact**: Empty or whitespace-only field names are normalized to 'product'
- **Real-world impact**: NEGLIGIBLE - Discord webhook fields always have non-empty names
- **Fix**: Add early return `if (!lower) return null;` at the start
